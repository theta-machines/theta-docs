## Overview
The motion controller is the hardware that communicates with the trajectory planning software. The motion controller's responsibility is to process instructions queued by the trajectory planner and generate the electrical signals needed to control a machine. In addition, the motion controller handles other hard real-time tasks like PID control and handling limit switch conditions Since all numerically controlled machines use similar hardware, a single motion control architecture can be used to control many types of machines.

The full motion control device is called an *RTMC card*. RTMC cards are circuit boards that combine an *RTMC core* with necessary memory and I/O devices. This relationship is similar to that of a *graphics card* and a *graphics processing unit*.

RTMC cores are defined by an instruction set architecture (ISA). This ISA is an abstract model of the core, and serves as the interface between the hardware device and software. 

## Architectures
Currently, there is one RTMC core architecture: the [*T0 ISA*](/architecture/rtmc/motion-controller/t0-isa). This architecture is for research and development purposes. It is not intended to define a fully functional machine controller.
